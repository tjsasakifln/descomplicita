# Story MS-001.1: Abstraction Layer — DataSourceClient + NormalizedRecord

**Epic:** MS-001 — Multi-Source Integration
**Status:** Ready
**Story Points:** 5
**Prioridade:** P0 — Critica

---

## Story

**As a** desenvolvedor do BidIQ,
**I want** uma interface abstrata `DataSourceClient` com schema normalizado,
**so that** eu possa integrar multiplas fontes de licitacao sem alterar o pipeline existente de filtragem, LLM e Excel.

---

## Story Context

**Existing System Integration:**
- Integrates with: `backend/pncp_client.py` (acoplamento direto ao PNCP)
- Technology: Python, httpx, ThreadPoolExecutor
- Touch points: `pncp_client.py`, `main.py`, `filter.py`, `schemas.py`

**Problema Atual:**
- `PNCPClient` e um cliente concreto sem interface abstrata
- `_normalize_item()` faz normalizacao inline especifica para formato PNCP
- `main.py` instancia `PNCPClient()` diretamente — impossivel trocar/adicionar fontes
- `filter.py` depende de campos PNCP-specific (`objetoCompra`, `valorTotalEstimado`)

**Solucao:**
- Criar pacote `backend/sources/` com ABC `DataSourceClient`
- Criar `NormalizedRecord` como schema unificado para todas as fontes
- Refatorar `PNCPClient` como `PNCPSource(DataSourceClient)` sem quebrar funcionalidade
- Adaptar `filter.py` para operar sobre `NormalizedRecord`

---

## Acceptance Criteria

### Interface Abstrata

1. `DataSourceClient` ABC criado em `backend/sources/base.py` com metodos:
   - `async def fetch_records(query: SearchQuery) -> List[NormalizedRecord]`
   - `def normalize(raw: dict) -> NormalizedRecord`
   - `source_name: str` (propriedade)
   - `is_healthy() -> bool` (health check)

2. `NormalizedRecord` dataclass criado em `backend/sources/base.py` com campos:
   - `id: str` (hash unico)
   - `source: str` (nome da fonte primaria)
   - `sources: List[str]` (todas as fontes onde apareceu)
   - `numero_licitacao: str`
   - `objeto: str`
   - `orgao: str`
   - `cnpj_orgao: str`
   - `uf: str`
   - `municipio: str`
   - `valor_estimado: Optional[float]`
   - `modalidade: str`
   - `modalidade_codigo: Optional[int]`
   - `data_publicacao: Optional[datetime]`
   - `data_abertura: Optional[datetime]`
   - `status: Optional[str]`
   - `url_edital: Optional[str]`
   - `url_fonte: Optional[str]`
   - `raw_data: dict`

### PNCP Adapter

3. `PNCPSource` criado em `backend/sources/pncp_source.py` implementando `DataSourceClient`
4. Toda logica de fetch, retry, cache e paginacao migrada do `pncp_client.py` existente
5. `_normalize_item()` adaptado para retornar `NormalizedRecord`

### Compatibilidade

6. `filter.py` atualizado para operar sobre campos de `NormalizedRecord` (mapeamento de campos)
7. `main.py` atualizado para instanciar `PNCPSource` em vez de `PNCPClient`
8. Todos os testes existentes continuam passando sem modificacao
9. Comportamento identico ao atual — zero regressao funcional

### Configuracao

10. `config.py` expandido com `SOURCES_CONFIG` dict contendo configuracao por fonte:
    - `enabled: bool`
    - `base_url: str`
    - `auth: Optional[dict]`
    - `rate_limit_rps: int`
    - `timeout: int`
    - `priority: int`
