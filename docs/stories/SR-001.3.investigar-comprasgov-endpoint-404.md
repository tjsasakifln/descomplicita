# Story SR-001.3: Investigar e Corrigir Endpoint Compras.gov.br (404)

**Epic:** SR-001 — Search Resilience
**Status:** Done
**Story Points:** 3
**Prioridade:** P1 — Alta

---

## Story

**As a** desenvolvedor do pipeline de busca,
**I want** que o endpoint do Compras.gov.br esteja atualizado,
**so that** a fonte contribua dados ao inves de gerar erros 404 silenciosos.

---

## Story Context

**Existing System Integration:**
- Config: `backend/config.py:78-84` — `SOURCES_CONFIG["comprasgov"]`
- Source: `backend/sources/comprasgov_source.py`
- Testes: `backend/tests/test_comprasgov_source.py`

**Problema:**
Todas as requests para `dadosabertos.compras.gov.br/licitacoes/v1/licitacoes.json` retornam `404 Resource Not Found`. O endpoint pode ter sido deprecado ou migrado para nova versao.

```
ERROR | sources.comprasgov_source | Compras.gov.br pagination error at offset 0:
  Client error '404 Resource Not Found' for url
  'https://dadosabertos.compras.gov.br/licitacoes/v1/licitacoes.json?...'
```

---

## Acceptance Criteria

### Investigacao

1. Verificar documentacao atual: https://dadosabertos.compras.gov.br
2. Testar se endpoint migrou para `/v2/` ou outro path
3. Verificar se o servico foi descontinuado em favor do PNCP (consolidacao federal)

### Resolucao (uma das opcoes)

**Opcao A — Endpoint migrou:**
4. Atualizar `base_url` e paths em `config.py` e `comprasgov_source.py`
5. Atualizar testes
6. Validar com request real

**Opcao B — Endpoint deprecado:**
4. Desabilitar source: `"enabled": False` em `config.py`
5. Adicionar comentario com data e motivo da desabilitacao
6. Criar issue futura se alternativa for identificada
