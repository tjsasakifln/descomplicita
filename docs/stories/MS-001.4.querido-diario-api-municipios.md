# Story MS-001.4: Querido Diario — API Diarios Oficiais Municipais

**Epic:** MS-001 — Multi-Source Integration
**Status:** Ready
**Story Points:** 5
**Prioridade:** P1

---

## Story

**As a** usuario do Descomplicita buscando licitacoes municipais,
**I want** que a busca inclua dados do Querido Diario (OKBR),
**so that** eu encontre licitacoes publicadas em diarios oficiais de 350+ municipios que podem nao estar no PNCP.

---

## Story Context

**Existing System Integration:**
- Depends on: MS-001.1 (DataSourceClient interface)
- Technology: Python, httpx, REST API (FastAPI backend do Querido Diario)
- Touch points: `backend/sources/querido_diario_source.py` (novo), `config.py`

**Fonte:**
- **API:** `https://queridodiario.ok.org.br/api/gazettes`
- **Docs:** `https://docs.queridodiario.ok.org.br/pt-br/latest/utilizando/api-publica.html`
- **GitHub API:** `https://github.com/okfn-brasil/querido-diario-api`
- **GitHub Scrapers:** `https://github.com/okfn-brasil/querido-diario`
- **Auth:** Nenhuma
- **Formato:** JSON
- **Cobertura:** 350+ municipios brasileiros

**Valor Unico:**
- Cobertura municipal massiva de diarios oficiais
- Busca full-text em publicacoes oficiais
- Captura licitacoes de municipios que ainda nao migraram para PNCP
- Projeto open-source ativo com cobertura crescente

**Desafio:**
- Dados sao texto livre (publicacao do diario oficial), nao dados estruturados
- Requer parsing/regex para extrair informacoes de licitacao do texto

## Acceptance Criteria

### Client Implementation

1. `QueridoDiarioSource` criado em `backend/sources/querido_diario_source.py` implementando `DataSourceClient`
2. Conecta a API `https://queridodiario.ok.org.br/api/gazettes`
3. Parametros de busca:
   - `querystring`: termos de busca (keywords do setor)
   - `territory_ids`: codigos IBGE dos municipios (filtra por UF)
   - `published_since`: data inicial (YYYY-MM-DD)
   - `published_until`: data final (YYYY-MM-DD)
   - `size`: itens por pagina
   - `offset`: paginacao
4. Paginacao implementada via `offset` + `size`

### Mapeamento UF -> Territory IDs

5. Tabela de mapeamento UF -> lista de codigos IBGE dos municipios cobertos pelo Querido Diario
6. Metodo `get_territory_ids(uf: str) -> List[str]` que retorna municipios disponiveis por estado
7. Fallback: se UF nao tem municipios cobertos, retornar lista vazia (sem erro)

### Parsing de Texto Livre

8. `_extract_from_gazette(gazette: dict) -> List[NormalizedRecord]` que:
   - Busca padroes de licitacao no texto do excerto (pregao, edital, tomada de precos, etc.)
   - Extrai numero da licitacao via regex (ex: "Pregão Eletrônico nº 001/2026")
   - Extrai valor estimado via regex (ex: "R$ 150.000,00")
   - Extrai orgao do campo `territory_name` + contexto do texto
   - Extrai modalidade do texto (pregao, concorrencia, dispensa, etc.)
9. Se parsing falhar, incluir registro com `objeto = excerto[:200]` e campos opcionais como None
10. Campo `url_fonte` aponta para URL do diario oficial original

### Normalizacao

11. Campos mapeados para `NormalizedRecord`:
    - `territory_name` -> `municipio`
    - UF derivado do `territory_id` (2 primeiros digitos IBGE)
    - Texto extraido -> `objeto`
    - Regex extraido -> `numero_licitacao`, `valor_estimado`, `modalidade`
    - `date` -> `data_publicacao`
    - `url` -> `url_fonte`
12. `source` definido como `"querido_diario"`

### Resiliencia

13. Rate limiting de 5 req/s
14. Timeout de 20s
15. Municipios sem cobertura no QD nao geram erro (skip silencioso)

### Testes

16. Testes unitarios com mock (gazette com dados parseados, gazette sem match, UF sem cobertura)
17. Teste de parsing com exemplos reais de texto de diario oficial

## Technical Notes

### API Request Example

```
GET https://queridodiario.ok.org.br/api/gazettes?
  querystring=licitação+pregão+uniforme
  &territory_ids=3550308  (Sao Paulo-SP)
  &published_since=2026-02-01
  &published_until=2026-03-06
  &size=50
  &offset=0
```

### Response Format

```json
{
  "total_gazettes": 42,
  "gazettes": [
    {
      "territory_id": "3550308",
      "territory_name": "São Paulo",
      "date": "2026-02-15",
      "url": "https://...",
      "edition": "123",
      "is_extra_edition": false,
      "excerpts": [
        "PREGÃO ELETRÔNICO Nº 001/2026 ... aquisição de uniformes escolares ... R$ 250.000,00 ..."
      ],
      "txt_url": "https://..."
    }
  ]
}
```

### Regex Patterns para Parsing

```python
REGEX_NUMERO = r'(?:Pregão|Concorrência|Tomada|Dispensa|Edital)\s*(?:Eletrônico|Presencial)?\s*[Nn]?[ºo.]?\s*(\d{1,5}[/-]\d{4})'
REGEX_VALOR = r'R\$\s*([\d.,]+)'
REGEX_MODALIDADE = r'(Pregão\s*Eletrônico|Pregão\s*Presencial|Concorrência|Tomada\s*de\s*Preços?|Dispensa)'
```

### Codigos IBGE por UF (primeiros 2 digitos)

| UF | Codigo | UF | Codigo |
|----|--------|-----|--------|
| SP | 35 | RJ | 33 |
| MG | 31 | BA | 29 |
| PR | 41 | RS | 43 |
| PE | 26 | CE | 23 |
