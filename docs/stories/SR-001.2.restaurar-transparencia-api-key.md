# Story SR-001.2: Restaurar Transparencia API Key no Railway

**Epic:** SR-001 — Search Resilience
**Status:** Ready
**Story Points:** 1
**Prioridade:** P0 — Critica

---

## Story

**As a** operador do sistema,
**I want** que a API do Portal da Transparencia esteja autenticada corretamente,
**so that** a fonte retorne dados complementares ao PNCP.

---

## Story Context

**Existing System Integration:**
- Config: `backend/config.py:86-93` — `SOURCES_CONFIG["transparencia"]`
- Source: `backend/sources/transparencia_source.py`
- Env var: `TRANSPARENCIA_API_KEY`
- Portal: https://portaldatransparencia.gov.br/api-de-dados

**Problema:**
Todas as requests para `api.portaldatransparencia.gov.br` retornam `401 Unauthorized`. A env var `TRANSPARENCIA_API_KEY` esta ausente ou expirada no ambiente Railway.

```
ERROR | sources.transparencia_source | Transparencia API auth failure: 401
```

Afeta todas as 7 UFs simultaneamente.

---

## Acceptance Criteria

1. Verificar se a API key existe no Railway (`railway variables`)
2. Se ausente ou expirada, gerar nova chave no Portal da Transparencia (cadastro gratuito)
3. Configurar no Railway: `railway variables set TRANSPARENCIA_API_KEY=<chave>`
4. Validar com request manual: `curl -H "chave-api-dados: <key>" "https://api.portaldatransparencia.gov.br/api-de-dados/licitacoes?pagina=1"`
5. Confirmar nos logs Railway que requests retornam 200

---

## Technical Notes

### Configuracao atual

```python
# config.py:86-93
"transparencia": {
    "enabled": True,
    "base_url": "https://api.portaldatransparencia.gov.br",
    "auth": {"type": "api_key", "header": "chave-api-dados", "env_var": "TRANSPARENCIA_API_KEY"},
    "rate_limit_rps": 3,
    "timeout": 30,
    "priority": 3,
},
```

### Passos para obter API key

1. Acessar https://portaldatransparencia.gov.br/api-de-dados
2. Clicar em "Cadastre-se" (gratuito)
3. Confirmar email
4. Copiar a chave gerada
5. `railway variables set TRANSPARENCIA_API_KEY=<chave>`
