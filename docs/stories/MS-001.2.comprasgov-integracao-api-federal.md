# Story MS-001.2: Compras.gov.br — Integracao API Federal Historica

**Epic:** MS-001 — Multi-Source Integration
**Status:** Ready
**Story Points:** 5
**Prioridade:** P0 — Critica

---

## Story

**As a** usuario do BidIQ buscando licitacoes federais,
**I want** que a busca inclua dados do Compras.gov.br (SIASG),
**so that** eu encontre licitacoes federais historicas e dados enriquecidos com CATMAT/CATSER que nao estao no PNCP.

---

## Story Context

**Existing System Integration:**
- Depends on: MS-001.1 (DataSourceClient interface)
- Technology: Python, httpx, REST API
- Touch points: `backend/sources/comprasgov_source.py` (novo), `config.py`

**Fonte:**
- **API Swagger:** `https://dadosabertos.compras.gov.br/swagger-ui/index.html`
- **API Legacy:** `http://compras.dados.gov.br/docs/home.html`
- **Contratos API:** `https://contratos.comprasnet.gov.br/api/docs`
- **GitHub Docs:** `https://github.com/gestaogovbr/Compras-API-doc`
- **Auth:** Nenhuma
- **Formato:** JSON, XML, CSV
- **Cobertura:** Federal (SIASG) — dados pre e pos 2023

**Valor Unico:**
- Dados historicos federais desde antes do PNCP (pre-2023)
- Categorizacao CATMAT (materiais) e CATSER (servicos) detalhada
- Dados de fornecedores e atas de pregao
- Complementa PNCP com profundidade historica

---

## Acceptance Criteria

### Client Implementation

1. `ComprasGovSource` criado em `backend/sources/comprasgov_source.py` implementando `DataSourceClient`
2. Conecta a API `dadosabertos.compras.gov.br` endpoint `/licitacoes/v1/licitacoes.json`
3. Suporta parametros de busca: `data_inicial`, `data_final`, `uf`, `modalidade`
4. Paginacao implementada conforme formato da API (offset-based ou page-based)

### Normalizacao

5. `normalize()` mapeia campos Compras.gov para `NormalizedRecord`:
   - `numero` -> `numero_licitacao`
   - `objeto` -> `objeto`
   - `orgao` -> `orgao`
   - `uasg` -> contexto adicional em `raw_data`
   - `valor_estimado` -> `valor_estimado`
   - `modalidade_licitacao` -> `modalidade`
   - `data_publicacao` -> `data_publicacao`
   - `situacao_licitacao` -> `status`
6. `source` definido como `"comprasgov"`
7. `url_fonte` gera link para pagina da licitacao no Compras.gov.br

### Resiliencia

8. Retry com exponential backoff (reutilizar `RetryConfig` existente)
9. Rate limiting de 5 req/s (configuravel)
10. Timeout de 20s por request
11. `is_healthy()` implementado com ping ao endpoint base

### Configuracao

12. Entrada em `SOURCES_CONFIG` com `enabled: True` por padrao
13. Logs estruturados com `source=comprasgov` para rastreabilidade

### Testes

14. Testes unitarios com mock da API (minimo 3 cenarios: sucesso, paginacao, erro)
15. Teste de normalizacao validando mapeamento de campos

## Technical Notes

### Endpoints Principais

| Endpoint | Dados |
|----------|-------|
| `/licitacoes/v1/licitacoes.json` | Lista de licitacoes com filtros |
| `/contratos/v1/contratos.json` | Contratos firmados |
| `/fornecedores/v1/fornecedores.json` | Fornecedores cadastrados |
| `/materiais/v1/materiais.json` | Catalogo CATMAT |

### Parametros de Query

```
?data_publicacao_min=DD/MM/YYYY
&data_publicacao_max=DD/MM/YYYY
&uf=SP
&modalidade=5  (pregao eletronico)
&offset=0
&limit=500
```

### Mapeamento de Modalidades

| Compras.gov | Equivalente PNCP |
|-------------|-----------------|
| 1 - Convite | Nao disponivel no PNCP |
| 2 - Tomada de Precos | Nao disponivel no PNCP |
| 3 - Concorrencia | 4, 5 |
| 5 - Pregao | 6, 7 |
| 6 - Dispensa | 8 |
