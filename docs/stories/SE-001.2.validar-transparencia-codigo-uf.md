# Story SE-001.2: Validar e Corrigir Parametro `codigoUF` na Transparencia

**Epic:** SE-001 — Search Effectiveness
**Status:** Ready
**Story Points:** 2
**Prioridade:** P0 — Critica

---

## Story

**As a** usuario buscando licitacoes federais,
**I want** que a fonte Transparencia retorne dados corretamente para todos os estados,
**so that** eu receba licitacoes federais complementares ao PNCP.

---

## Story Context

**Existing System Integration:**
- Arquivos: `backend/sources/transparencia_source.py`
- Testes: `backend/tests/test_transparencia_source.py` (se existir)
- Dependencia: nenhuma

**Problema:**
Em `transparencia_source.py:267`, o parametro `codigoUF` recebe a sigla do estado (ex: `"SP"`, `"RJ"`) diretamente do `SearchQuery.ufs`. A API do Portal da Transparencia pode esperar codigos IBGE numericos (ex: `35` para SP, `33` para RJ). Se estiver recebendo siglas invalidas, a API retorna silenciosamente 0 resultados em vez de erro, mascarando o problema.

**Investigacao necessaria:**
1. Consultar documentacao oficial da API (`https://portaldatransparencia.gov.br/api-de-dados`)
2. Testar ambos os formatos via curl/httpx
3. Implementar mapeamento se necessario

---

## Acceptance Criteria

### 1. Formato do `codigoUF` validado

- Documentacao oficial da API consultada e formato correto confirmado
- Teste manual com ao menos 3 UFs (SP, RJ, MG) validando retorno de dados

### 2. Mapeamento implementado (se necessario)

- Se API espera codigo IBGE numerico: mapeamento de sigla para codigo IBGE
- Mapeamento completo para todos os 27 estados + DF
- Mapeamento como constante em `transparencia_source.py` ou em modulo compartilhado

### 3. Tratamento de erro para UF invalida

- Se API retorna erro para UF invalida, log com nivel WARNING
- Se API retorna vazio silenciosamente, log informativo com UF usada

### 4. Testes

- Teste unitario validando mapeamento de todas as 27 UFs
- Teste de integracao (mock) validando que `codigoUF` e enviado no formato correto
- Se formato atual estiver correto: teste documentando a validacao

---

## Tarefas Tecnicas

1. Consultar documentacao da API Portal da Transparencia para formato `codigoUF`
2. Testar manualmente com curl: sigla vs codigo IBGE
3. Se necessario, criar constante `UF_TO_IBGE_CODE` com mapeamento
4. Modificar `fetch_records()` para converter UF antes de enviar
5. Adicionar log informativo quando UF e convertida
