# Story MS-001.6: Multi-Source Orchestrator — Busca Paralela + Deduplicacao

**Epic:** MS-001 — Multi-Source Integration
**Status:** Ready
**Story Points:** 8
**Prioridade:** P0 — Critica

---

## Story

**As a** usuario do Descomplicita,
**I want** que a busca consulte todas as fontes habilitadas em paralelo e remova duplicatas automaticamente,
**so that** eu receba uma lista unificada e limpa de licitacoes de multiplas fontes sem ver o mesmo item repetido.

---

## Story Context

**Existing System Integration:**
- Depends on: MS-001.1 (abstraction layer), MS-001.2 (Compras.gov)
- Optional deps: MS-001.3, MS-001.4, MS-001.5 (fontes adicionais)
- Technology: Python, asyncio, concurrent processing
- Touch points: `backend/sources/orchestrator.py` (novo), `main.py`, `schemas.py`, `job_store.py`

**Problema:**
- Com 5 fontes, buscar sequencialmente multiplicaria a latencia por 5x
- O mesmo processo licitatorio pode aparecer no PNCP + Compras.gov + Portal Transparencia
- Sem dedup, usuario veria duplicatas no Excel

**Solucao:**
- Orquestrador que busca em paralelo com timeout por fonte
- Deduplicacao por chave composta (CNPJ + numero + ano)
- Degradacao graciosa: se uma fonte falhar, as demais entregam resultado

---

## Acceptance Criteria

### Orquestrador

1. `MultiSourceOrchestrator` criado em `backend/sources/orchestrator.py`
2. Recebe lista de `DataSourceClient` habilitados via `SOURCES_CONFIG`
3. `search_all(query: SearchQuery) -> OrchestratorResult` executa todas as fontes em paralelo
4. Timeout individual por fonte (configuravel, default 30s)
5. Se uma fonte falhar/timeout, as demais continuam normalmente
6. Log de erro para fontes que falharam (nao propaga excecao)

### Resultado Unificado

7. `OrchestratorResult` contem:
   - `records: List[NormalizedRecord]` — lista unificada e deduplicada
   - `source_stats: Dict[str, SourceStats]` — stats por fonte:
     - `total_fetched: int`
     - `after_dedup: int`
     - `elapsed_ms: int`
     - `status: "success" | "error" | "timeout"`
     - `error_message: Optional[str]`
   - `dedup_removed: int` — total de duplicatas removidas
   - `sources_used: List[str]` — fontes que retornaram com sucesso

### Deduplicacao

8. Chave de deduplicacao: `hash(normalize(cnpj_orgao) + normalize(numero_licitacao) + ano)`
9. Quando duplicata detectada:
   - Manter registro com mais campos preenchidos (mais completo)
   - Agregar `sources[]` com todas as fontes onde apareceu
   - Preferir PNCP como `source` primario (fonte canonica)
10. Fallback: se `cnpj_orgao` ou `numero_licitacao` ausente, usar `hash(normalize(objeto[:100]) + uf + data_publicacao)`
11. Dedup nao deve exceder 50ms para 10.000 registros

### Integracao com Pipeline Existente

12. `main.py` atualizado: substituir `PNCPClient().fetch_all()` por `orchestrator.search_all()`
13. Resultado do orquestrador alimenta `filter_batch()` normalmente
14. `schemas.py` atualizado com campos `sources_used`, `source_stats`, `dedup_removed` no `BuscaResult`
15. `job_store.py` progress tracking atualizado com info de fontes

### Progress Tracking

16. Progresso reporta quantas fontes completaram: `sources_completed / sources_total`
17. Fase "fetching" mostra `"Buscando em N fontes..."` em vez de `"Buscando em X/Y estados"`

### Feature Flags

18. Cada fonte pode ser habilitada/desabilitada via `SOURCES_CONFIG[source].enabled`
19. Variavel de ambiente `ENABLED_SOURCES` (comma-separated) override para deploy gradual
20. Default: apenas PNCP habilitado (rollout progressivo)

## Technical Notes

### Arquitetura de Execucao

```python
class MultiSourceOrchestrator:
    async def search_all(self, query: SearchQuery) -> OrchestratorResult:
        tasks = []
        for source in self.enabled_sources:
            task = asyncio.create_task(
                self._fetch_with_timeout(source, query)
            )
            tasks.append((source.source_name, task))

        results = await asyncio.gather(*[t for _, t in tasks], return_exceptions=True)

        all_records = []
        source_stats = {}
        for (name, _), result in zip(tasks, results):
            if isinstance(result, Exception):
                source_stats[name] = SourceStats(status="error", error=str(result))
            else:
                all_records.extend(result.records)
                source_stats[name] = result.stats

        deduped, dedup_count = self._deduplicate(all_records)
        return OrchestratorResult(records=deduped, source_stats=source_stats, dedup_removed=dedup_count)
```

### Algoritmo de Deduplicacao

```python
def _deduplicate(self, records: List[NormalizedRecord]) -> Tuple[List, int]:
    seen = {}  # key -> NormalizedRecord
    duplicates = 0

    # Ordenar por prioridade da fonte (PNCP primeiro)
    records.sort(key=lambda r: self._source_priority(r.source))

    for record in records:
        key = self._dedup_key(record)
        if key in seen:
            # Merge: agregar sources, manter mais completo
            seen[key].sources.append(record.source)
            self._merge_fields(seen[key], record)
            duplicates += 1
        else:
            seen[key] = record

    return list(seen.values()), duplicates
```
