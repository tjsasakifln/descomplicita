# Story SE-001.5: Adicionar Endpoint PNCP `/atas` para Atas de Registro de Preco

**Epic:** SE-001 — Search Effectiveness
**Status:** Ready
**Story Points:** 5
**Prioridade:** P2 — Media

---

## Story

**As a** usuario buscando oportunidades de fornecimento publico,
**I want** que a busca inclua Atas de Registro de Preco publicadas no PNCP,
**so that** eu identifique oportunidades de adesao a atas vigentes (carona).

---

## Story Context

**Existing System Integration:**
- Arquivos: `backend/pncp_client.py`, `backend/sources/pncp_source.py`
- Testes: `backend/tests/test_pncp_client.py`
- Dependencia: SE-001.1 (SearchQuery com search_terms)

**Problema:**
O PNCP tem dois endpoints principais para dados de contratacao:
1. `/contratacoes/publicacao` — licitacoes (JA implementado)
2. `/atas` — atas de registro de preco (NAO implementado)

Atas de Registro de Preco sao contratos vigentes que permitem **adesao por outros orgaos** ("carona"). Para setores como saude e alimentos, atas representam volume significativo de oportunidades — um fornecedor pode aderir a uma ata existente sem participar de nova licitacao.

**Dados disponibilizados pelo endpoint `/atas`:**
- Numero da ata
- Orgao gerenciador
- Itens com precos registrados
- Vigencia (data inicio/fim)
- Fornecedores vencedores
- CNPJ e UF

---

## Acceptance Criteria

### 1. PNCPClient suporta endpoint `/atas`

- Novo metodo `fetch_atas()` em `PNCPClient`
- Parametros: data_inicial, data_final, ufs, palavra_chave (opcional)
- Paginacao identica a `fetch_page()` (tamanhoPagina=50)
- Rate limiting e retry compartilhados com endpoint existente

### 2. Normalizacao de atas para NormalizedRecord

- Atas normalizadas para `NormalizedRecord` com campo `tipo="ata_registro_preco"`
- Campos mapeados: numero, orgao, cnpj, uf, valor, vigencia, itens
- URL do edital/ata no PNCP

### 3. PNCPSource busca ambos os endpoints

- `fetch_records()` busca `/contratacoes/publicacao` E `/atas` em paralelo
- Resultados combinados e deduplicados
- Flag de configuracao para habilitar/desabilitar busca de atas

### 4. Frontend distingue atas de licitacoes

- Campo `tipo` propagado ate o frontend
- Atas exibidas com badge visual diferenciado (ex: "Ata RP")

### 5. Testes

- Testes unitarios para `fetch_atas()` com mocks
- Teste de normalizacao de ata para NormalizedRecord
- Teste de integracao: busca retorna mix de licitacoes + atas

---

## Tarefas Tecnicas

1. Investigar documentacao do endpoint `/atas` na API PNCP
2. Implementar `fetch_atas()` em `PNCPClient`
3. Implementar normalizacao de ata para `NormalizedRecord`
4. Modificar `PNCPSource.fetch_records()` para buscar atas em paralelo
5. Adicionar campo `tipo` a `NormalizedRecord` em `base.py`
6. Propagar `tipo` no pipeline ate o Excel e frontend
7. Adicionar testes
8. Validar com busca real no PNCP

## Riscos

| Risco | Mitigacao |
|-------|----------|
| Endpoint `/atas` pode ter estrutura diferente | Investigar Swagger do PNCP antes de implementar |
| Volume de atas pode ser alto | Limitar por palavraChave e data |
| Atas vencidas poluem resultados | Filtrar por vigencia (data_fim >= hoje) |
