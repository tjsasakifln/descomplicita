# Story SR-001.5: Investigar e Corrigir TCE-RJ Endpoint (404)

**Epic:** SR-001 — Search Resilience
**Status:** Done
**Story Points:** 2
**Prioridade:** P1 — Alta

---

## Story

**As a** desenvolvedor do pipeline de busca,
**I want** que o endpoint do TCE-RJ esteja atualizado,
**so that** dados de compras diretas do RJ complementem os resultados do PNCP.

---

## Story Context

**Existing System Integration:**
- Config: `backend/config.py:102-109` — `SOURCES_CONFIG["tce_rj"]`
- Source: `backend/sources/tce_rj_source.py`
- Testes: `backend/tests/test_tce_rj_source.py`

**Problema:**
O endpoint do TCE-RJ retorna `404 Not Found` para compras diretas, seguido de timeout no orchestrator.

```
ERROR | sources.tce_rj_source | TCE-RJ compras-diretas pagination error at offset 0:
  Client error '404 Not Found' for url
  'https://dados.tcerj.tc.br/api/v1/compras-diretas?...'
ERROR | sources.orchestrator | Source tce_rj timed out
```

---

## Acceptance Criteria

### Investigacao

1. Verificar documentacao: https://dados.tcerj.tc.br
2. Testar se endpoint `/api/v1/compras-diretas` ainda existe
3. Verificar se o path mudou ou se a API foi descontinuada
4. Verificar se requer autenticacao que nao existia antes

### Resolucao (uma das opcoes)

**Opcao A — Endpoint migrou:**
5. Atualizar `base_url` e paths em `config.py` e `tce_rj_source.py`
6. Atualizar testes
7. Validar com request real

**Opcao B — Endpoint deprecado:**
5. Desabilitar source: `"enabled": False` em `config.py`
6. Adicionar comentario com data e motivo da desabilitacao
7. Criar issue futura se alternativa for identificada

---

```python
# config.py:102-109

```

| Arquivo | Mudanca possivel |
|---------|-----------------|
| `backend/config.py:102-109` | Atualizar `base_url` ou `enabled: False` |
| `backend/sources/tce_rj_source.py` | Atualizar URLs/paths |
| `backend/tests/test_tce_rj_source.py` | Atualizar testes |

---

## Resolution

**Option B adopted** — Endpoint deprecated.

The TCE-RJ `/api/v1/compras-diretas` endpoint returns `404 Not Found` since ~2026-03.
Source disabled in `config.py` with `"enabled": False` and deprecation comment referencing SR-001.5.
Test added (`test_deprecated_sources_disabled`) to assert all deprecated sources remain disabled.

### Changes Made

| File | Change |
|------|--------|
| `backend/config.py:107` | `"enabled": False` + deprecation comment |
| `backend/tests/test_sources_config.py` | Added `test_deprecated_sources_disabled` |
