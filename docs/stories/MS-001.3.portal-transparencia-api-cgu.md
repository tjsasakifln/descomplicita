# Story MS-001.3: Portal da Transparencia — API CGU + Listas de Sancao

**Epic:** MS-001 — Multi-Source Integration
**Status:** Ready
**Story Points:** 5
**Prioridade:** P1

---

## Story

**As a** usuario do BidIQ avaliando oportunidades de licitacao,
**I want** que a busca inclua dados do Portal da Transparencia e verifique listas de sancao,
**so that** eu tenha uma visao mais completa das licitacoes federais e identifique riscos em orgaos/fornecedores sancionados.

---

## Story Context

**Existing System Integration:**
- Depends on: MS-001.1 (DataSourceClient interface)
- Technology: Python, httpx, REST API
- Touch points: `backend/sources/transparencia_source.py` (novo), `config.py`

**Fonte:**
- **API:** `https://portaldatransparencia.gov.br/api-de-dados`
- **Catalogo:** `https://www.gov.br/conecta/catalogo/apis/portal-da-transparencia-do-governo-federal`
- **Auth:** API key gratuita (registro por email)
- **Formato:** JSON
- **Cobertura:** Poder Executivo Federal

**Valor Unico:**
- Listas de sancao CEIS (empresas impedidas), CNEP (penalidades), CEPIM (entidades impedidas)
- Cross-reference independente de dados federais
- Dados de convenios e despesas para contexto

---

## Acceptance Criteria

### Client Implementation

1. `TransparenciaSource` criado em `backend/sources/transparencia_source.py` implementando `DataSourceClient`
2. Conecta a API com header `chave-api-dados: {API_KEY}`
3. Endpoint principal: `/api-de-dados/licitacoes`
4. Suporta filtros: `dataInicial`, `dataFinal`, `codigoUF`, `modalidadeLicitacao`
5. Paginacao implementada conforme API (pagina 1-indexed)

### Listas de Sancao (Diferencial)

6. Metodo auxiliar `check_sanctions(cnpj: str) -> SanctionResult` que consulta:
   - CEIS: `/api-de-dados/ceis` — empresas impedidas de licitar
   - CNEP: `/api-de-dados/cnep` — entidades com penalidades
7. `SanctionResult` retorna: `is_sanctioned: bool`, `sanctions: List[dict]`
8. Resultado de sancao adicionado como campo extra em `NormalizedRecord.raw_data`

### Normalizacao

9. Campos mapeados para `NormalizedRecord`:
   - `numero` -> `numero_licitacao`
   - `objeto` -> `objeto`
   - `orgaoVinculado.nome` -> `orgao`
   - `orgaoVinculado.cnpj` -> `cnpj_orgao`
   - `uf` -> `uf`
   - `municipio` -> `municipio`
   - `valorLicitacao` -> `valor_estimado`
   - `modalidadeLicitacao` -> `modalidade`
   - `dataAbertura` -> `data_abertura`
   - `situacaoLicitacao` -> `status`
10. `source` definido como `"transparencia"`

### Configuracao e Seguranca

11. `API_KEY` lido de variavel de ambiente `TRANSPARENCIA_API_KEY`
12. Entrada em `SOURCES_CONFIG` com `enabled: True`
13. Rate limiting de 3 req/s (API mais restritiva)
14. Timeout de 30s (API pode ser lenta)

### Testes

15. Testes unitarios com mock (sucesso, auth failure, sancao encontrada, sancao limpa)

## Technical Notes

### Registro da API Key

1. Acessar `https://portaldatransparencia.gov.br/api-de-dados`
2. Clicar em "Cadastre-se" (gratuito)
3. Informar email e dados basicos
4. Receber API key por email
5. Configurar `TRANSPARENCIA_API_KEY` no `.env`

### Endpoints Relevantes

| Endpoint | Dados |
|----------|-------|
| `/api-de-dados/licitacoes` | Licitacoes federais |
| `/api-de-dados/contratos` | Contratos federais |
| `/api-de-dados/ceis` | Empresas impedidas |
| `/api-de-dados/cnep` | Penalidades aplicadas |
| `/api-de-dados/cepim` | Entidades impedidas |

### Header de Autenticacao

```
GET /api-de-dados/licitacoes?dataInicial=01/01/2026&dataFinal=06/03/2026
Headers:
  chave-api-dados: {TRANSPARENCIA_API_KEY}
  Accept: application/json
```
