# Story SR-001.4: Corrigir Querido Diario JSON Parse Error

**Epic:** SR-001 — Search Resilience
**Status:** Ready
**Story Points:** 2
**Prioridade:** P1 — Alta

---

## Story

**As a** desenvolvedor do pipeline de busca,
**I want** que o Querido Diario retorne dados validos ou falhe graciosamente,
**so that** erros de parsing nao mascarem problemas silenciosamente.

---

## Story Context

**Existing System Integration:**
- Config: `backend/config.py:94-100` — `SOURCES_CONFIG["querido_diario"]`
- Source: `backend/sources/querido_diario_source.py`
- Testes: `backend/tests/test_querido_diario_source.py`

**Problema:**
O source Querido Diario falha com erro de JSON parsing: `Expecting value: line 1 column 1 (char 0)`. Isso indica que a API esta retornando corpo vazio, HTML, ou redirecionamento em vez de JSON.

```
ERROR | sources.querido_diario_source | Querido Diario pagination error at offset 0:
  Expecting value: line 1 column 1 (char 0)
```

---

## Acceptance Criteria

### Investigacao

1. Testar manualmente: `curl -s -i "https://queridodiario.ok.org.br/api/gazettes?..."`
2. Verificar se a API mudou o content-type ou requer headers adicionais
3. Verificar se ha rate limiting retornando HTML em vez de JSON

### Parsing defensivo

4. Validar `Content-Type: application/json` antes de chamar `response.json()`
5. Se content-type nao for JSON, logar os primeiros 200 chars do body para debug
6. Retornar lista vazia em vez de crashar quando body e invalido

### Resolucao

7. Se endpoint mudou: atualizar URL/headers em `querido_diario_source.py`
8. Se API foi deprecada: desabilitar source com justificativa
