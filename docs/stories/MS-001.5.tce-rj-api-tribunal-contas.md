# Story MS-001.5: TCE-RJ ‚Äî API Tribunal de Contas do Estado do RJ

**Epic:** MS-001 ‚Äî Multi-Source Integration
**Status:** Ready
**Story Points:** 3
**Prioridade:** P1

---

## Story

**As a** usuario do Descomplicita buscando licitacoes no Rio de Janeiro,
**I want** que a busca inclua dados do TCE-RJ,
**so that** eu encontre licitacoes e compras diretas de todos os municipios do RJ em uma unica fonte.

---

## Story Context

**Existing System Integration:**
- Depends on: MS-001.1 (DataSourceClient interface)
- Technology: Python, httpx, REST API
- Touch points: `backend/sources/tce_rj_source.py` (novo), `config.py`

**Fonte:**
- **API Docs:** `https://dados.tcerj.tc.br/api/v1/docs`
- **Auth:** Nenhuma (endpoints publicos)
- **Formato:** JSON
- **Cobertura:** Todos os municipios do Rio de Janeiro + orgaos estaduais

**Valor Unico:**
- Fonte unica para TODOS os municipios do RJ (92 municipios)
- Inclui compras diretas (dispensas, inexigibilidades, adesoes a atas)
- Dados de licitacoes + contratos do Tribunal de Contas
- API REST bem documentada

---

## Acceptance Criteria

### Client Implementation

1. `TCERJSource` criado em `backend/sources/tce_rj_source.py` implementando `DataSourceClient`
2. Conecta a API `https://dados.tcerj.tc.br/api/v1/`
3. Endpoints utilizados:
   - `/licitacoes` ‚Äî licitacoes publicadas
   - `/compras-diretas` ‚Äî dispensas e inexigibilidades
4. Suporta filtros de data, municipio/orgao
5. Paginacao implementada conforme API

### Filtro por UF

6. Source so e ativado quando `"RJ"` esta nas UFs selecionadas
7. Se UF nao inclui RJ, `fetch_records()` retorna lista vazia imediatamente
8. Sem overhead para buscas que nao incluem RJ

### Normalizacao

9. Campos mapeados para `NormalizedRecord`:
   - Numero da licitacao/compra -> `numero_licitacao`
   - Objeto -> `objeto`
   - Orgao/entidade -> `orgao`
   - CNPJ do orgao -> `cnpj_orgao`
   - `"RJ"` -> `uf` (fixo)
   - Municipio -> `municipio`
   - Valor estimado -> `valor_estimado`
   - Modalidade -> `modalidade`
   - Data publicacao -> `data_publicacao`
   - Situacao -> `status`
10. `source` definido como `"tce_rj"`
11. Compras diretas normalizadas com `modalidade = "Dispensa"` ou `"Inexigibilidade"`

### Resiliencia

12. Retry com exponential backoff
13. Rate limiting de 5 req/s
14. Timeout de 20s

### Testes

15. Testes unitarios com mock (licitacao, compra direta, UF != RJ skip)

## Technical Notes

### Endpoints Documentados

| Endpoint | Dados |
|----------|-------|
| `/api/v1/licitacoes` | Licitacoes de todos os municipios RJ |
| `/api/v1/compras-diretas` | Dispensas, inexigibilidades, adesoes |
| `/api/v1/contratos` | Contratos firmados |

### Consideracao de Escopo

Este source e especifico para RJ. O padrao pode ser replicado para outros TCEs:
- TCE-SP (`transparencia.tce.sp.gov.br/apis`)
- TCE-PE (`sistemas.tce.pe.gov.br/DadosAbertos/`)

A mesma interface `DataSourceClient` permite adicionar TCEs futuros facilmente.
