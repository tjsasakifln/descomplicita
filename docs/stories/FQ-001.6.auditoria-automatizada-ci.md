# Story FQ-001.6: Auditoria Automatizada como CI Quality Gate

**Epic:** FQ-001 — Eliminacao de Falsos Positivos
**Status:** Draft
**Story Points:** 3
**Prioridade:** P2 — Media
**Dependencia:** FQ-001.5

---

## Story

**As a** desenvolvedor do BidIQ,
**I want** que um quality gate automatizado valide a taxa de falsos positivos a cada PR que modifique a logica de filtragem,
**so that** regressoes de qualidade sejam detectadas antes do merge.

---

## Story Context

**Existing System Integration:**
- Integrates with: `backend/scripts/audit_all_sectors.py`, `backend/scripts/audit_false_positives.py`
- Technology: Python, GitHub Actions
- Follows pattern: scripts de auditoria ja existem mas sao executados manualmente
- Touch points: `.github/workflows/`, `backend/scripts/`

**Problema:** Hoje a auditoria de FPs e manual e ad-hoc. Nao ha mecanismo para prevenir regressoes quando exclusoes ou keywords sao modificados.

---

## Acceptance Criteria

### Functional Requirements

1. Script de auditoria refatorado para saida machine-readable (JSON + exit code)
2. Exit code 0 quando FP < threshold; exit code 1 quando FP >= threshold
3. Threshold configuravel por setor (padrao: 3% por setor, 1% global)
4. Relatorio markdown gerado automaticamente como artefato

### Integration Requirements

5. GitHub Action roda automaticamente em PRs que modifiquem `sectors.py`, `filter.py`, ou `pncp_client.py`
6. Roda como job opcional (nao bloqueia merge em caso de timeout da API PNCP)
7. Resultados postados como comentario no PR

### Quality Requirements

8. Tempo de execucao < 5 minutos (limitar amostra a 500 itens se necessario)
9. Funciona offline com dados cached quando API PNCP estiver indisponivel

## Design Tecnico

### Script de Auditoria Refatorado

```python
# backend/scripts/audit_quality_gate.py
def main():
    items = fetch_or_cache_pncp_data(max_items=500)
    results = analyze_all_sectors(items)

    # Check thresholds
    failures = []
    for r in results:
        if r["fp_rate"] > SECTOR_THRESHOLD:
            failures.append(f"{r['sector']}: {r['fp_rate']:.1%} > {SECTOR_THRESHOLD:.1%}")

    # Output
    report = generate_markdown_report(results)
    with open("audit_report.md", "w") as f:
        f.write(report)

    if failures:
        print("QUALITY GATE FAILED:")
        for f in failures:
            print(f"  - {f}")
        sys.exit(1)

    print("QUALITY GATE PASSED")
    sys.exit(0)
```

### GitHub Action

```yaml
name: Filter Quality Audit
on:
  pull_request:
    paths:
      - 'backend/sectors.py'
      - 'backend/filter.py'
      - 'backend/pncp_client.py'

jobs:
  audit:
    runs-on: ubuntu-latest
    continue-on-error: true  # nao bloqueia merge
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - run: pip install requests
      - run: cd backend && python scripts/audit_quality_gate.py
      - uses: actions/upload-artifact@v4
        with:
          name: audit-report
          path: backend/audit_report.md
```

### Cache de Dados PNCP

Para evitar dependencia da API PNCP em CI:
